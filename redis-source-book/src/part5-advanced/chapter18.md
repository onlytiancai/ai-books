# 第18章：Virtual Memory 概述

Virtual Memory (VM) 是 Redis 2.0.0 引入的特性，用于将不常访问的数据交换到磁盘。

## 18.1 VM 设计理念

### 18.1.1 目的

Redis 是内存数据库，所有数据都存储在内存中。VM 的目的是：

1. 突破内存限制，存储更多数据
2. 利用磁盘空间存储冷数据
3. 保持热数据在内存中快速访问

### 18.1.2 工作原理

```
┌──────────────────────────────────────────────────────────────┐
│                         内存                                  │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐                        │
│  │  热数据  │ │  热数据  │ │  指针   │ ──► 磁盘位置          │
│  └─────────┘ └─────────┘ └─────────┘                        │
└──────────────────────────────────────────────────────────────┘
                                        │
                                        v
┌──────────────────────────────────────────────────────────────┐
│                         磁盘                                  │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐           │
│  │  冷数据  │ │  冷数据  │ │  冷数据  │ │  冷数据  │           │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘           │
└──────────────────────────────────────────────────────────────┘
```

## 18.2 VM 配置

```c
// redis.conf
vm-enabled yes
vm-swap-file /tmp/redis.swap
vm-max-memory 1000000000  // 最大内存使用量
vm-page-size 32           // 页大小（字节）
vm-pages 1000000          // 总页数
vm-max-threads 4          // 最大 IO 线程数
```

## 18.3 VM 状态

```c
#define REDIS_VM_MEMORY 0     // 数据在内存中
#define REDIS_VM_SWAPPED 1    // 数据已交换到磁盘
#define REDIS_VM_SWAPPING 2   // 正在交换到磁盘
#define REDIS_VM_LOADING 3    // 正在从磁盘加载
```

## 18.4 为何被移除

Redis 2.4 之后 VM 被移除，原因：

1. **复杂度高**：增加了代码复杂度
2. **性能问题**：磁盘 IO 影响响应时间
3. **维护成本**：难以维护和调试
4. **替代方案**：
   - Redis Cluster 分片
   - 使用更大的内存
   - 操作系统虚拟内存

## 18.5 小结

本章简要介绍了 Redis 2.0.0 的 Virtual Memory 特性：

1. **设计目的**：突破内存限制
2. **工作原理**：冷热数据分离
3. **历史地位**：已移除的特性
4. **移除原因**：复杂度和性能问题

VM 是 Redis 发展过程中的一个尝试，虽然最终被移除，但其设计思想值得了解。
