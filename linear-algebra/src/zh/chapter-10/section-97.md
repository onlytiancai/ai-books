### 97. 推荐系统与低秩模型（填充缺失项）

推荐系统（如 Netflix、Amazon 或 Spotify 所使用的系统）构建于一个核心原理之上：用户的偏好可以通过隐藏在大型稀疏数据中的低维结构来捕捉。线性代数为我们提供了揭示和利用这些结构的工具，尤其是通过**低秩模型（Low-Rank Models）**。

#### 偏好矩阵

我们从用户 - 物品矩阵 $R \in \mathbb{R}^{m \times n}$ 开始：

- 行代表用户（Users）。
- 列代表物品（Items，如电影、书籍、歌曲）。
- 条目 $R_{ij}$ 存储评分（例如 1–5 星）或交互行为（点击、购买）。

在实践中，大多数条目是缺失的——用户通常只对一小部分物品进行评分。**核心挑战在于：预测这些缺失的条目。**

#### 为什么是低秩结构？

尽管矩阵 $R$ 的尺寸很大，但它通常接近一个低秩近似：

$$
R \approx U V^T
$$

- $U \in \mathbb{R}^{m \times k}$：用户因子矩阵。
- $V \in \mathbb{R}^{n \times k}$：物品因子矩阵。
- $k \ll \min(m, n)$。

在这里，每个用户和每个物品都在一个共享的**潜在特征空间（Latent Feature Space）**中表示。

- 示例：对于电影，潜在维度可能捕捉“动作片 vs 爱情片”、“旧片 vs 新片”或“主流 vs 独立制作”。
- 用户在此空间中的偏好向量与物品的特征向量相互作用，生成预测评分。

这种分解解释了相关性：如果你喜欢电影 A 和 B，而电影 C 共享相似的潜在特征，系统会预测你也会喜欢 C。

> **注意**：这里的 $k$ 被称为“秩”（Rank）。选择较小的 $k$ 意味着我们假设用户的偏好是由少数几个潜在因素决定的，这有助于去除数据中的噪声。

#### 奇异值分解（SVD）方法

如果 $R$ 是完整的（没有缺失条目），我们可以计算 SVD：

$$
R = U \Sigma V^T.
$$

- 保留前 $k$ 个奇异值以形成秩 -$k$ 近似。
- 这捕捉了用户偏好中的主导模式。
- 几何视角：将庞大的数据云投影到一个更小的 $k$ 维子空间上，在那里结构更加清晰。

但现实数据是不完整的。这导致了**矩阵补全（Matrix Completion）**问题。

下面是一个使用 Python `numpy` 演示完整矩阵 SVD 低秩近似的简单示例：

```python
import numpy as np

# 假设一个完整的用户 - 物品矩阵 (5 用户 x 4 物品)
R_complete = np.array([
    [5, 4, 1, 0],
    [4, 5, 2, 1],
    [1, 2, 5, 4],
    [0, 1, 4, 5],
    [3, 3, 2, 2]
])

# 进行奇异值分解
U, Sigma, VT = np.linalg.svd(R_complete, full_matrices=False)

# 保留前 k=2 个奇异值进行低秩近似
k = 2
Sigma_k = np.diag(Sigma[:k])
U_k = U[:, :k]
V_k = VT[:k, :]

# 重构矩阵
R_approx = U_k @ Sigma_k @ V_k

print("原始矩阵:\n", R_complete)
print("秩 -2 近似矩阵:\n", np.round(R_approx, 2))
```

#### 矩阵补全

矩阵补全试图通过假设低秩来推断 $R$ 的缺失条目。优化问题如下：

$$
\min_{X} \ \text{rank}(X) \quad \text{s.t. } X_{ij} = R_{ij} \text{ for observed entries}.
$$

由于最小化秩是 NP 难问题，实际算法通常最小化**核范数（Nuclear Norm，奇异值之和）**或使用交替最小化：

- 随机初始化 $U, V$。
- 迭代地固定其中一个求解另一个。
- 收敛到符合观测评分的低秩分解。

#### 交替最小二乘法（ALS）

ALS 是一种标准方法：

1. 固定 $V$，通过最小二乘法求解 $U$。
2. 固定 $U$，通过最小二乘法求解 $V$。
3. 重复直到收敛。

每个子问题都是简单的线性回归，可以通过正规方程或 QR 分解求解。

#### 随机梯度下降（SGD）

另一种方法：将每个观测评分视为一个训练样本。通过最小化平方误差来更新潜在向量：

$$
\ell = (R_{ij} - u_i^T v_j)^2.
$$

沿梯度迭代调整用户向量 $u_i$ 和物品向量 $v_j$。这种方法能很好地扩展到海量数据集，因此在实践中非常常见。

#### 正则化

为了防止过拟合：

$$
\ell = (R_{ij} - u_i^T v_j)^2 + \lambda (\|u_i\|^2 + \|v_j\|^2).
$$

- 正则化会收缩因子， discouraging 极端值。
- 几何上，它使潜在向量保持在特征空间中的一个合理球体内。

> **提示**：正则化参数 $\lambda$ 控制模型的复杂度。$\lambda$ 过大可能导致欠拟合（所有预测值趋同），$\lambda$ 过小可能导致过拟合（模型记住了噪声）。

#### 冷启动问题

- 新用户：没有评分记录，$u_i$ 未知。解决方案：使用人口统计特征或要求用户进行少量初始评分。
- 新物品：同样，物品需要侧边信息（元数据、标签）来生成初始潜在向量。

这就是混合模型将矩阵分解与基于内容的特征相结合的地方。

#### 示例：电影评分

想象有 1,000 名用户和 5,000 部电影。

- 原始 $R$ 矩阵有 500 万个条目，但每个用户只评分了约 50 部电影。
- 秩 $k = 20$ 的矩阵补全恢复了一个稠密近似。
- 每个用户由 20 个潜在“口味”因子表示；每部电影由 20 个潜在“主题”因子表示。
- 预测：用户向量与电影向量的点积。

#### 超越评分：隐式反馈

在实践中，系统通常缺乏显式评分。相反，它们使用：

- 浏览、点击、购买、跳过。
- 这些信号是间接的但数量丰富。
- 分解可以通过将交互视为加权观测来处理它们。

#### 与其他线性代数工具的联系

- **SVD（第 9 章）**：分解方法的骨干。
- **伪逆（第 9 章）**：在求解 ALS 中的小型回归子问题时很有用。
- **条件数（第 9 章）**：分解的稳定性取决于潜在因子的缩放比例。
- **PCA（第 96 章）**：PCA 本质上是低秩近似，因此 PCA 和推荐系统共享相同的数学基础。

#### 为什么这很重要

推荐系统个性化了现代互联网。每一个播放列表建议、书籍推荐或广告放置都是由隐藏在巨大稀疏矩阵中的线性代数驱动的。低秩模型表明，即使是不完整、嘈杂的数据也可以被利用来揭示偏好和行为模式。

#### 练习题解答与代码实现

以下我们将通过完整的 Python 代码来解答上述四个练习。我们将使用 `numpy` 来模拟矩阵分解过程。

```python
import numpy as np
from numpy.linalg import svd, norm

# ==========================================
# 准备数据：创建一个带有缺失值的小型矩阵
# ==========================================
# 假设 5 个用户，4 个物品。用 0 表示缺失（实际应用中通常用 NaN，这里为了计算方便用 0 掩码）
R_original = np.array([
    [5, 3, 0, 1],
    [4, 0, 0, 1],
    [1, 1, 0, 5],
    [0, 0, 4, 4],
    [3, 2, 0, 2]
])

# 创建掩码矩阵，1 表示观测值，0 表示缺失
mask = (R_original > 0).astype(float)

# 为了演示 SVD，我们需要先填充缺失值（例如用平均值填充）作为初始完整矩阵
row_mean = np.sum(R_original, axis=1) / np.sum(mask, axis=1)
R_filled = R_original.copy()
for i in range(R_filled.shape[0]):
    for j in range(R_filled.shape[1]):
        if R_filled[i, j] == 0:
            R_filled[i, j] = row_mean[i]

print("--- 练习 1: 使用 SVD 进行秩 -2 近似填充 ---")
# 1. 应用秩 -2 近似
U, Sigma, VT = svd(R_filled, full_matrices=False)
k = 2
Sigma_k = np.diag(Sigma[:k])
R_approx_svd = U[:, :k] @ Sigma_k @ VT[:k, :]

print("原始矩阵 (0 代表缺失):")
print(R_original)
print("\nSVD 秩 -2 填充后的矩阵:")
print(np.round(R_approx_svd, 2))

# ==========================================
# 练习 2: 实现 ALS 的一步
# ==========================================
print("\n--- 练习 2: 实现 ALS 的一步 (固定 V 更新 U) ---")

# 随机初始化 U 和 V (假设秩 k=2)
m, n = R_original.shape
k = 2
np.random.seed(42)
U_als = np.random.rand(m, k)
V_als = np.random.rand(n, k)

# ALS 一步：固定 V，求解 U
# 对于每个用户 i，求解 min || R_i - u_i^T V^T ||^2 (仅针对观测值)
# 简化演示：这里我们忽略掩码，直接对完整矩阵做最小二乘演示原理
# 实际 ALS 需要针对每个用户只使用其观测到的物品列
U_new = np.zeros_like(U_als)
for i in range(m):
    # 获取用户 i 观测到的物品索引
    observed_items = np.where(mask[i, :] == 1)[0]
    if len(observed_items) > 0:
        V_sub = V_als[observed_items, :] # 对应的物品因子
        R_sub = R_original[i, observed_items] # 对应的评分
        
        # 最小二乘解: u_i = (V_sub^T V_sub)^-1 V_sub^T R_sub
        # 注意：这里 R_sub 是行向量，公式需转置适配
        try:
            u_i = np.linalg.lstsq(V_sub, R_sub, rcond=None)[0]
            U_new[i, :] = u_i
        except:
            U_new[i, :] = U_als[i, :] # 如果无法求解，保持原值

print("更新后的用户因子矩阵 U (部分):")
print(np.round(U_new, 4))

# ==========================================
# 练习 3: 比较正则化
# ==========================================
print("\n--- 练习 3: 正则化的影响 ---")

# 定义损失函数
def calculate_loss(R, U, V, mask, lambda_reg=0.0):
    pred = U @ V.T
    error = (R - pred) * mask # 只计算观测值的误差
    mse = np.sum(error ** 2) / np.sum(mask)
    reg_term = lambda_reg * (np.sum(U**2) + np.sum(V**2))
    return mse + reg_term

# 无正则化
loss_no_reg = calculate_loss(R_original, U_new, V_als, mask, lambda_reg=0.0)
# 有正则化 (lambda = 0.1)
loss_reg = calculate_loss(R_original, U_new, V_als, mask, lambda_reg=0.1)

print(f"无正则化损失 (仅 MSE 部分): {loss_no_reg:.4f}")
print(f"有正则化损失 (含惩罚项): {loss_reg:.4f}")
print("提示：正则化会增加总损失函数的值，但能防止 U 和 V 的数值过大，提高泛化能力。")

# ==========================================
# 练习 4: 冷启动问题模拟
# ==========================================
print("\n--- 练习 4: 冷启动问题模拟 ---")

# 模拟一个新用户，没有任何评分记录
new_user_ratings = np.zeros(n) 
# 策略：使用全局平均向量或现有用户的平均因子作为初始猜测
avg_user_factor = np.mean(U_new, axis=0)

# 预测该新用户对所有物品的评分
pred_new_user = avg_user_factor @ V_als.T

print("新用户初始潜在因子 (使用现有用户平均):")
print(np.round(avg_user_factor, 4))
print("基于此因子预测的新用户评分:")
print(np.round(pred_new_user, 2))
print("注意：冷启动下预测往往趋向于平均值，直到收集到足够的新数据。")
```

**解答过程分析：**

1.  **SVD 填充**：
    *   **思路**：首先必须将缺失值暂时填充（例如用行均值），才能进行 SVD 分解。分解后截取前 $k$ 个奇异值重构矩阵，重构后的矩阵在原本缺失的位置会有数值，即为预测值。
    *   **结果**：代码展示了如何将稀疏矩阵转化为稠密的低秩近似矩阵。

2.  **ALS 一步实现**：
    *   **思路**：ALS 的核心是交替优化。当固定 $V$ 时，对于每个用户 $i$，问题变成了求解线性方程组 $V_{obs}^T u_i \approx R_{i, obs}$。
    *   **关键点**：必须只使用用户实际评分过的物品（`observed_items`）来构建方程组，否则会将缺失值误认为 0 分。代码中使用了 `np.linalg.lstsq` 来求解最小二乘问题。

3.  **正则化对比**：
    *   **思路**：在损失函数中加入 $\lambda (\|u_i\|^2 + \|v_j\|^2)$。
    *   **观察**：虽然正则化项会增加总的损失值（因为加了惩罚项），但它限制了向量的模长。在没有正则化时，模型可能会为了拟合噪声而使向量数值变得极大，导致在未见数据上表现糟糕。

4.  **冷启动模拟**：
    *   **思路**：对于没有任何数据的新用户，无法通过优化求解 $u_{new}$。
    *   **策略**：最简单的基线方法是使用所有现有用户因子的平均值（`np.mean(U, axis=0)`）。这意味着我们假设新用户具有“平均品味”。随着新用户产生少量交互，再通过梯度下降微调其向量。
